# Harness Coding Protocol

Make AI coding assistants read the right thing before they change your code.

Harness Coding Protocol is a **Repository AI Governance Starter Kit**.  
It turns `AGENTS.md`, `CLAUDE.md`, and `steering/` into one clear root entry layer for AI collaboration in any repository.

Works well with Claude Code, Codex, Cursor, Kiro, and other MCP-compatible tools.

## One-Line Value

Stop scattering AI rules across README files, IDE-private folders, and one-off instructions.  
Keep repository truth at the root so humans, AI agents, and IDE tooling all agree on what to read, what wins, and what a finished setup should look like.

## What You Get

- A clean root truth layer: `AGENTS.md` + `CLAUDE.md` + `steering/`
- An installable starter kit instead of loose templates
- Optional Cursor and Kiro mirrors without polluting repository truth
- One install command plus one validation command
- Two ready-to-read examples: `minimal` and `complete`

## What It Solves

- AI agents need a stable place to read repository facts
- Multi-tool setups become confusing when rules are duplicated across tool-specific folders
- Drift happens when README, templates, and installer behavior describe different paths

## Good Fit If

- You want AI collaboration rules without adopting a heavy framework
- You use more than one coding assistant or IDE workflow
- You want root governance and tool compatibility layers to stay separate
- You want something a new repo can adopt in minutes and an existing repo can adopt safely

## Core Model

| Layer | Path | Purpose |
|-------|------|---------|
| Facts | `AGENTS.md` | Verifiable repository facts: structure, commands, ports, endpoints, conventions |
| Protocol | `CLAUDE.md` | Decision priority, conflict resolution, RIPER-5, collaboration rules |
| Local overrides | `steering/*.md` | Extra guidance for specific paths, stacks, or tasks |

Tool-private folders are not repository truth.  
`.cursor/rules/` and `.kiro/steering/` are optional compatibility mirrors generated by the installers.

## 30-Second Start

```bash
bash scripts/apply-template.sh /your/project --with-cursor --with-kiro --example complete
node scripts/validate-template.mjs /your/project
```

After installation, the repository truth is always:

```text
AGENTS.md
CLAUDE.md
steering/
```

## 5-Minute Setup

### Option A: Recommended installer

```bash
bash scripts/apply-template.sh /your/project --with-cursor --with-kiro --example complete
```

Windows PowerShell:

```powershell
powershell -File scripts/apply-template.ps1 C:\your\project --with-cursor --with-kiro --example complete
```

The default file strategy is `--skip-existing`. Use `--overwrite` or `--backup` when needed.

### Option B: Manual copy

```bash
cp templates/root/AGENTS.md /your/project/AGENTS.md
cp templates/root/CLAUDE.md /your/project/CLAUDE.md
cp -R templates/steering /your/project/steering
```

### Option C: Validate the result

```bash
node scripts/validate-template.mjs /your/project
```

## Rule Priority

1. User instruction
2. Repository root `AGENTS.md`
3. Repository root `CLAUDE.md`
4. Matching `steering/*.md`
5. Tool adapter files
