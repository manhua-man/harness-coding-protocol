# Frontend Steering

> 适用于 Web / App 前端界面的局部规则。只在处理前端代码、交互、样式和体验问题时引用。

## Goals

- 优先保持交互清晰、加载迅速、状态可见
- 组件职责单一，避免在一个组件里同时堆叠数据获取、复杂状态机和大量表现逻辑
- 修改界面时优先贴合现有设计语言，而不是重做视觉体系

## Implementation Notes

- 优先复用已有组件、tokens、布局约束，再新增抽象
- 如果需要引入状态管理或副作用逻辑，优先放入 hooks / composables / stores，而不是塞进视图组件
- 可访问性至少覆盖语义标签、键盘可用性、焦点顺序和文本对比度
- 样式变更要兼顾移动端和桌面端，避免只在单一视口下验证

## Verification

- 提供最小可验证结果：截图、交互录屏或组件测试其一
- 改动涉及条件渲染、表单或异步加载时，补充对应测试或至少说明验证步骤
