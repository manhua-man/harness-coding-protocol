# Backend Steering

> 适用于服务端、数据层、接口和任务处理逻辑。只在处理后端代码时引用。

## Goals

- 优先保证行为正确、接口清晰、失败可诊断
- 控制器/路由层负责协议转换，业务逻辑放在 service / domain 层
- 依赖关系显式化，避免隐藏状态和难以替换的全局依赖

## Implementation Notes

- 新增接口时明确输入、输出、错误分支和权限边界
- 涉及数据库或缓存时，优先说明读写路径、事务边界与幂等性要求
- 只在确有必要时抽象公共层，避免为单点逻辑引入额外 indirection
- 日志记录要便于排障，但避免泄露敏感信息或制造重复噪音

## Verification

- 至少覆盖一个成功路径和一个失败路径
- 改动涉及持久化、队列、重试或并发时，补充针对性测试或明确手工验证步骤
