# TDD + Quality Bundle

## Purpose

This bundle supports test-first or test-driven work with a strong quality gate.

## Recommended Components

- Test planning
- Red-green-refactor workflow
- Linting or static checks
- Review pass for regression risk

## Good Fit For

- Core logic changes
- Bug fixes with regression risk
- Refactors that touch shared code
- Changes that should leave a clear test trail

## Boundary

- This bundle recommends quality practices; it does not force a specific test framework
- The repository should still document actual project commands in the root truth files
