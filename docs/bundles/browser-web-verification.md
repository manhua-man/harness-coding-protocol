# Browser / Web Verification Bundle

## Purpose

This bundle supports web and browser verification tasks where execution feedback matters.

## Recommended Components

- Browser automation
- Screenshot or visual inspection steps
- Interaction checks for forms, navigation, and state changes
- Optional network or console inspection

## Good Fit For

- UI smoke checks
- Web flow validation
- Layout verification
- Regression checks after frontend changes

## Boundary

- Browser verification is a support layer, not a substitute for root-truth documentation
- The bundle should describe explicit checks rather than implying unattended end-to-end certainty
